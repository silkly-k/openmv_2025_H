## 3. 自适应二值化 —— 非线性系统 (1:35 - 2:20)

**画面**：
- 在 OpenMV IDE 中展示 `src/detectors.py:21-40` 的 `binary_auto` 函数。
- 串行终端打印当前帧的 `mean` 和计算出的 `thr` 值（例如 `mean=148, thr=156`）。
- IDE 帧缓冲区切换到二值化图像（黑白两色，形状清晰）。
- 用手略微遮挡环境光，观察阈值数字自动下降，二值化效果仍稳定。

**旁白**：
「为了从灰度图中分割出形状，需要二值化。但固定阈值对光照敏感——如果环境光变暗，所有像素灰度都降低，固定阈值就会失效。
所以我们采用**自适应阈值**：代码在 `detectors.py:27-28`——先从感兴趣区域（ROI）取灰度均值 `mean`，然后加偏移量 `mean_offset=8`，再钳位到 `[min_thr=60, max_thr=210]` 范围。
当前帧的均值为 148，计算得到阈值 156。所有灰度 ≤156 的像素判为前景（黑色形状），反之为背景。

二值化是一个**非线性变换**：输入是 0-255 灰度，输出只有 0 或 255。其映射关系（阈值）取决于输入信号的统计值（均值），因此是**信号依赖的非线性系统**——这不能用一个固定的传递函数来描述。」

**演示动作**：
- 用手遮挡环境光 → 均值降到 ~90 → 阈值自动降到 ~98 → 图像仍然正确分割。
- （可对比：注释掉 `adaptive` 改用 `fixed_thr=90`，遮挡后全黑或全白）。

**代码展示**：
```python
# src/detectors.py:21-40 — binary_auto()
st = gray_img.get_statistics(roi=stats_roi)  # 统计中心ROI
mean_v = int(st.mean())
if BIN["adaptive"]:
    thr = int(clamp(mean_v + BIN["mean_offset"],  # mean+8
                    BIN["min_thr"], BIN["max_thr"]))  # [60, 210]
else:
    thr = BIN["fixed_thr"]  # 90
gray_img.binary([(thr, 255)])  # 二值化
```

## 4. 形态学滤波 —— 非线性滤波器 (2:20 - 3:00)

**画面**：
- 二值化后图像局部放大：可见纸张上微小灰尘产生的孤立白点。
- 在 IDE 中单步执行 `close(1)`，观察小噪点被填充。
- 高亮 `src/config.py:59-60`：`"open_n": 0, "close_n": 1`。

**旁白**：
「二值化后，图像可能有一些微小噪点。当前配置只用 **闭运算**（`close_n=1`，`open_n=0`），即先膨胀后腐蚀。
闭运算填充形状内部的小空洞，同时保形边缘不被破坏。与线性滤波（如均值滤波）不同，形态学操作基于结构元素的几何特性——它是**非线性的几何滤波器**，能在去噪的同时完美保持形状边缘。
从信号处理角度，这体现了非线性滤波在保持信号细节（边缘）上的优势——线性低通滤波会导致边缘模糊，而形态学滤波不会。」

**代码展示**：
```python
# src/detectors.py:35-38 — 形态学滤波
if BIN["open_n"] > 0:
    gray_img.open(BIN["open_n"])     # 先腐蚀后膨胀 → 去白点
if BIN["close_n"] > 0:
    gray_img.close(BIN["close_n"])   # 先膨胀后腐蚀 → 填黑洞
# 当前配置: open_n=0（跳过）, close_n=1（执行1次闭运算）
```

**演示动作**：
- 用铅笔在纸上轻轻点一个小黑点 → 闭运算把它填充掉。
- 提示如果需要消除白点（而非黑洞），把 `open_n` 改为 1。

## 5. 锚点检测与 A4 定位 —— 两级级联检测 (3:00 - 3:50)

**画面**：
- 切换到 `src/pipeline.py:24-29`：先调 `detect_anchor_blob`，再调 `detect_a4_blob`。
- 在 IDE 帧缓冲区上显示调试叠加层：中心十字 = 锚点，绿色方框 = A4 候选。
- 展示锚点检测的代码逻辑：在中心 ROI 中找黑色 blob（灰度范围 `[0, dark_hi]`，其中 `dark_hi = thr - 25`）。

**旁白**：
「接下来是 A4 纸定位，实际用了两级级联检测：
**第一级——锚点检测**（`detect_anchor_blob`）：在二值化图像中，A4 纸中央有一个预先画好的黑色实心圆（Anchor）。代码在中心 ROI 内寻找灰度在 `[0, thr-25]` 范围内的 blob，按面积和中心度打分选出最佳锚点。
**第二级——A4 检测**（`detect_a4_blob`）：利用锚点位置来辅助筛选 A4 候选。注意：二值化后纸张是**白色**的（255），所以这里寻找的是灰度 `[250, 255]` 范围内的 blob——不是找黑区域，是找白区域！

A4 候选按以下条件筛选：
- 宽高比 `w/h` 在 `[0.55, 0.90]`（A4 竖放约为 0.707 = 1/√2）
- 填充率 `pixels/box_area` 在 `[0.45, 0.98]`
- 位置偏移不超过图像中心 ROI 的 40%
如果有锚点命中（anchor_hit），候选评分加 500 分，大幅提升正确率。

这是一个**信号检测与估计**的经典问题：在噪声中定位已知尺寸的信号（A4 纸），用锚点作为辅助参考信号来降低虚警率。」

**代码展示**：
```python

anchor, anchor_candidates = detect_anchor_blob(bw, center_search_roi, thr)
# ... anchor 坐标用于辅助 A4 检测 ...
a4, a4_candidates = detect_a4_blob(bw, search_roi=center_search_roi, anchor=anchor)
```

## 6. 形状识别与尺寸测量 —— 降维与决策 (3:50 - 4:40)

**画面**：
- 切换到 `src/detectors.py:132-207` 的 `detect_shape_blobs` 函数。
- 串行终端打印每个候选形状的 `fill_ratio` 和 `shape_type`。
- IDE 帧缓冲区上用不同颜色框出各形状候选（红色=最优，白色=其他）。
- 展示 `detectors.py:61-77` 的距离与尺寸估算公式。

**旁白**：
「A4 定位完成后，在 A4 内部缩小边距（`inner_margin_px=16` 像素）得到形状搜索 ROI。
`detect_shape_blobs` 在此 ROI 中找黑色 blob（灰度 `[0, thr-25]`），提取两个特征：
1. **长宽比** `w/h`：与 1.0 的偏差不超过 `shape_aspect_tol=0.40`
2. **填充率** `pixels/(w*h)`：
   - 三角形：0.32 ~ 0.62（面积约为外接矩形的一半）
   - 圆形：0.58 ~ 0.84（面积约为外接矩形的 π/4 ≈ 0.785）
   - 矩形：0.84 ~ 1.00（几乎填满外接矩形）

把每个 blob 从高维像素空间映射到二维特征空间 `(aspect, fill_ratio)`，再通过阈值判定形状类别——这就是**降维 + 决策边界**的经典方法。

距离估算（`detectors.py:52-58`）用针孔模型：
`d = (focal_px × A4宽度210mm) / 纸张像素宽度 × distance_gain + distance_offset`
即 `(560 × 210) / paper_width_px × 0.884 + 38`

尺寸估算（`detectors.py:61-76`）用比例法：
`size_mm = (shape_px / paper_width_px) × 210mm × size_gain + size_offset`
圆形取宽高均值，方形取长边。

这些参数在 `config.py:42-50` 的 `MEASURE` 字典里，目前是工程估算值，需要标定后替换。」

**演示动作**：

**旁白**：
「OpenMV 将测量结果编码为简洁的 ASCII 字符串，通过 UART3 发送。实际协议（`main.py:102`）是：
`D:452.3,X:48.5,S:RECT\n`
三个字段分别表示距离(mm)、尺寸(mm)、形状类型。

串行通信将多个数值转换为**时间序列信号**：起始位、8 数据位、停止位构成每帧 10 bit，保证接收端时钟同步。波特率 115200 即每秒 115200 个符号位。

注意 `main.py` 第 13 行有一个全局开关 `DEBUG_TO_PC = True`——设为 `True` 时，信息同时通过 `print()` 发到 IDE 终端和通过 UART 发到 STM32；设为 `False` 时只发 UART，减少 USB 开销。这在调试阶段非常有用。」

**代码展示**：
```python
# main.py:98-102 — UART 发送
d_val = result.get("d_mm", 0)
x_val = result.get("x_mm", 0)
shape = _shape_from_status(result.get("status", ""))
uart.write("D:%.1f,X:%.1f,S:%s\n" % (d_val, x_val, shape))
```

**注意事项**：

- 若串口无输出：确认 `DEBUG_TO_PC` 不影响 UART 发送（UART 总是发的）。

### 8.1 映射到图形元素 (5:20 - 5:55)

**画面**：

- 展示 STM32 端伪代码（解析 `D:...X:...S:...` 格式）。

**旁白**：
「STM32 在 UART 中断中逐字符接收，检测到换行符 `\n` 后，用 `strtok` 按逗号或冒号解析出 D、X、S 的数值。
通过 LVGL 更新界面元素：
- 距离值 → 进度条长度（线性映射）
- 尺寸值 → 数字标签
- 形状类型字符 → 切换对应图标（RECT/CIRCLE/TRI）

这类似于**数模转换**：离散的数字量被重构为连续的人眼视觉信号。LVGL 刷新率设为 20 Hz（50ms 周期），高于人眼视觉暂留频率（~10 Hz），画面感觉平滑连续——这正是**采样与重构**思想的应用。」

### 8.3 量化误差的可视化 (6:20 - 6:40)

**画面**：
- 缓慢移动 A4 纸，使距离从 400 mm 增加到 410 mm。若代码中对距离取整到 5 mm，进度条呈阶跃式跳动。

**旁白**：
「如果我们在 STM32 端将距离值除以 5 取整再显示，进度条会以 5 mm 步长阶跃变化。这是一种**幅度量化**，量化步长（LSB）为 5 mm。
步长太大 → 人眼看到明显跳跃（量化误差大）。
步长太小 → 显示抖动（噪声导致的 LSB 翻转）。
实际工程中需要根据噪声水平选择合理的量化步长。这就是量化噪声与有效位数的直观体现。」

### 9.3 调试技巧 (7:35 - 7:45)

**旁白**：
「实验中可能遇到的问题及排查方法：
- **二值化异常**：打开 IDE 直方图看灰度分布，调整 `mean_offset` 或 `min_thr/max_thr`。
- **A4 检测不到**：检查 `a4_aspect_min/max`（A4 竖放约 0.707）、`a4_fill_min/max`；确认锚点圆清晰。
- **形状识别错误**：用 IDE 帧缓冲区放大观察 blob 的 fill_ratio，按实际打印质量微调 `rect_fill_min` 等阈值。
- **STM32 收不到数据**：用串口调试助手在 OpenMV TX 引脚监听，确认有数据输出 → 再查线序和共地。
- **处理时间过长**：在终端看 `PROC=xxxus`，超过 50ms 时可考虑降低分辨率或简化形态学操作。
核心思想：**分模块调试**，逐级确认信号流每个节点的输出是否符合预期。」
